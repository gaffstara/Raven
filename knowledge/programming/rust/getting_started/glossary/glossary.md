# Rust Glossary

Summary

Common Rust terms and concepts useful for developers and knowledge systems.

Definitions

- `Ownership`: the rule that each value has a single owner responsible for cleanup.
- `Borrowing`: using references to access data without taking ownership.
- `Lifetime`: the period during which a reference remains valid.
- `Trait`: a set of required methods implemented by types.
- `MIR`: Mid-level Intermediate Representation used by the compiler.
- `Crate`: a compilation unit or library in Rust.

Notes

Use this glossary as a reference for terminology when reading Rust documentation or code.
